# ☕ AXIOM_CAFFEINE // Asistente de Extracción Molecular

Un asistente interactivo de extracción e hidratación en tiempo real diseñado para amantes del café de especialidad y profesionales en entornos de alto rendimiento. Esta aplicación web elimina la necesidad de pesados vídeos precargados o interfaces estáticas mediante un motor logicial nativo que asiste al usuario en la preparación perfecta de su dosis diaria de cafeína.

---

## 🛠️ Especificaciones Tecnológicas (Pila Core)

Siguiendo principios estrictos de optimización de memoria y velocidad móvil, la aplicación se empaqueta en un **único archivo unificado (`index.html`)** libre de dependencias o frameworks pesados:

* **HTML5 Semántico:** Estructura atómica y limpia para tarjetas de recetas y telemetría.
* **CSS3 Avanzado (Aceleración GPU):**
    * **Glassmorphism Puro:** Componentes de interfaz translúcidos basados en hardware mediante capas de filtrado (`backdrop-filter: blur()`).
    * **Modulación Lumínica Fluida:** Variables globales nativas (`:root`) para transiciones instantáneas entre los modos Día (Claro) y Noche (Oscuro).
* **Vanilla JavaScript (ES6+ Strict Core):** Gestión síncrona de eventos táctiles, hilos de cronometración y manipulación reactiva del DOM.
* **Librerías Ligeras Integradas (vía CDN):**
    * `Lenis`: Desplazamiento táctil y scroll suavizado de alta densidad.
    * `GSAP`: Control elástico de vistas y transiciones elásticas simuladas (*Spring Physics*).
    * `Lucide Icons`: Carga síncrona de iconografía minimalista.
    * `Canvas-Confetti`: Disparador procedural de recompensas al finalizar la infusión.

## 🏎️ Arquitectura de Motores Integrados

### 1. Curvas de Extracción (Rust `no_std` Logic)
El objeto de datos unificado (*Asset Pipeline*) actúa como un búfer de datos que segmenta los ratios exactos de agua/café de especialidad (V60, Espresso, Prensa Francesa, Moka). El motor calcula de forma determinista la cantidad de miligramos de cafeína liberados y el agua de hidratación real agregada en base al volumen molecular de cada método.

### 2. Simulador de Llenado Líquido Cinético
En lugar de animaciones precargadas, la taza central renderiza dinámicamente un plano de fluido procedural. Al iniciar el temporizador, un algoritmo mapea el tiempo transcurrido con el porcentaje de llenado del lienzo, aplicando una animación de oleaje continua (`liquid-wave`) que responde síncronamente al reloj.

### 3. Reordenación Telemétrica (AutoAnimate Mock)
Al finalizar la preparación de la taza, los datos de telemetría e hidratación del usuario se actualizan instantáneamente en la memoria RAM. El script evalúa el peso del impacto biométrico y reordena las tarjetas del panel de control de forma nativa sin generar parpadeos ni recargas de pantalla.
